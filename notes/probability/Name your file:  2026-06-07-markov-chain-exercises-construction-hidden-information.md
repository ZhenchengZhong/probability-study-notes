## Exercise 8.1: Constructing a Markov Chain on Sequence Space

Exercise 8.1 asks us to prove Theorem 8.1 for finite state space S by constructing a probability measure directly on the sequence space S^∞.

The main idea is to treat each infinite sequence

ω = (u_1, u_2, u_3, ...)

as one possible path of the Markov chain.

For a finite initial path

(u_1, u_2, ..., u_n),

we assign probability

μ_{u_1} p_{u_1 u_2} p_{u_2 u_3} ... p_{u_{n-1} u_n}.

This formula means:

- first choose the initial state u_1 according to μ,
- then move from u_1 to u_2 according to p_{u_1 u_2},
- then from u_2 to u_3 according to p_{u_2 u_3},
- and so on.

This gives a probability rule on cylinder sets, which are events determined by finitely many coordinates. Since S is finite, the same compactness-style argument used before Theorem 2.3 shows that the probability is countably additive on the cylinder field.

After extending this probability measure to the σ-field generated by cylinders, the coordinate process

X_n(ω) = z_n(ω)

becomes the required Markov chain. The time index is shifted by 1 because the coordinate functions are written as z_1, z_2, ..., while Theorem 8.1 usually starts from X_0.

The key point is that a Markov chain can be constructed by assigning probabilities to finite paths in a consistent way.

## Exercise 8.2: Hidden Information Can Destroy the Markov Property

Exercise 8.2 considers independent and identically distributed Bernoulli random variables Y_0, Y_1, ..., with

P(Y_n = 1) = p,

P(Y_n = 0) = q = 1 - p,

where p ≠ q.

Define

X_n = Y_n + Y_{n+1} mod 2.

Then X_n only records whether Y_n and Y_{n+1} are equal or different.

Specifically,

X_n = 0 if Y_n = Y_{n+1},

X_n = 1 if Y_n ≠ Y_{n+1}.

Although X_{n-1} and X_{n+1} are independent, this does not imply that X_0, X_1, ... is a Markov chain.

The reason is that knowing X_n does not reveal the actual hidden values of Y_n and Y_{n+1}. For example, X_1 = 0 tells us only that

Y_1 = Y_2,

but it does not tell us whether both are 0 or both are 1.

When p ≠ q, this hidden distinction matters. The earlier value X_0 can still help us infer the hidden value of Y_2, and Y_2 affects X_2.

So the past still contains useful information after the present X_1 is known. This violates the Markov property.

The central lesson is:

independence between X_{n-1} and X_{n+1} is not the same as the Markov property.

## Exercise 8.3: A Function of a Markov Chain Need Not Be Markov

Exercise 8.3 asks for an example showing that a function of a Markov chain does not necessarily remain a Markov chain.

Exercise 8.2 gives a natural example.

Define

Z_n = (Y_n, Y_{n+1}).

Then Z_0, Z_1, ... is a Markov chain, because if we know

Z_n = (a, b),

then the next state must have the form

Z_{n+1} = (b, c),

where c is a new independent Bernoulli variable.

Now define

f(a, b) = a + b mod 2.

Then

f(Z_n) = Y_n + Y_{n+1} mod 2 = X_n.

But Exercise 8.2 showed that X_0, X_1, ... is not a Markov chain.

Therefore, Z_n is Markov, but f(Z_n) is not Markov.

The main reason is that f compresses the original state and loses information. Once information is lost, the earlier past may become useful again for predicting the future.

## Key Takeaways

Exercise 8.1 shows how to construct a Markov chain by assigning probabilities to finite paths on sequence space.

Exercise 8.2 shows that a process can fail to be Markov even if variables two steps apart are independent.

Exercise 8.3 shows that applying a function to a Markov chain can destroy the Markov property.

The common theme is information. A process is Markov only when the current state contains enough information to predict the next state. If the current state is too compressed, the past may still matter.
